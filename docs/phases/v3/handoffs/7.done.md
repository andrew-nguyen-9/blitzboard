**Gotchas for dependents (5/8/13)**

  (no session/service-role needed). Decrypt with the pipeline/service-role + `CREDENTIAL_VAULT_KEY`;
  owner can read but not decrypt the ciphertext. No `phone_encrypted` if `CREDENTIAL_VAULT_KEY` unset.
- **Env vars**: `NEXT_PUBLIC_HCAPTCHA_SITE_KEY` (client) + `HCAPTCHA_SECRET` (server) — unset → math
  fallback, signups still work. **Vercel BotID is BLOCKED/gated**: `verifyBotId()` is a no-op stub
  (deploy-only, needs `npm i botid` + dashboard Bot Management); upgrade path in the file's comment.
- **Rate-limit** is in-memory per-instance — swap for Upstash/KV if multi-instance. Route is `/api/auth/signup`.
