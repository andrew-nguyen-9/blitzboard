# v3 per-epic handoff files

Auto-generated by the orchestrator (`docs/phases/v3/V3_ORCHESTRATOR.md`, step 4).

Two file kinds:

- `<epic-id>.md` — **brief** (orchestrator writes up front): epic spec, Definition-of-Done
  bar, blocker policy, branch/commit rules, local repo pointers. An agent reads only its own
  brief — it never loads the full v3 spec.
- `<epic-id>.done.md` — **outcome note** (agent writes on finish): ≤15 lines — what shipped,
  new/changed files, key decisions, gotchas for dependents. The orchestrator injects an
  epic's upstream `.done.md` paths into dependent agents per the dependency map in
  `../V3_ORCHESTRATOR.md`. Note bodies never enter the main chat.

These files double as durable checkpoints: if the Ralph loop dies mid-run, a fresh
orchestrator re-reads `prd.json` + this directory (briefs + outcome notes) and resumes
without re-deriving anything.

Source spec: `../V3_RALPH_HANDOFF.md`.
