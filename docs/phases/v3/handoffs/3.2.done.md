# Epic 3.2 — Shared tooltip primitive — DONE

**Shipped:** One opaque, theme-aware tooltip primitive, reused on the homepage deck and `PlayerTable.tsx`.

**Files**
- `frontend/components/Tooltip.tsx` (new) — the primitive.
- `frontend/lib/playerTooltip.ts` + `.test.ts` (new) — pure null-safe row-summary builder + unit test.
- `frontend/components/PlayerTable.tsx` — per-row scouting tooltip (`decorative`, `side="bottom"`).
- `frontend/app/page.tsx` — deck "seven tools, one spine" info tooltip.

**Component** `Tooltip` (default export, `@/components/Tooltip`)
Props: `content: ReactNode`, `id?`, `side?: "top"|"bottom"` (default top), `decorative?` (aria-hidden when content duplicates host data), `className?`.
Usage: render it as a child of an ancestor marked **`group`** that is a positioning context (`relative`, or already `absolute`).

**Decisions / gotchas for dependents (Epic 2 deck, Epic 11 player page)**
- CSS-only enter/leave via `group-hover`/`group-focus-within`; no JS state. Bubble is `pointer-events-none`, so overlaying the next row never traps — it clears on row change (verified by Playwright: hovered row opacity 1, prior row →0).
- Reduced motion needs no extra CSS — globals.css already collapses all `transition-duration` for both the OS query and `[data-motion="reduce"]`; the fade just becomes instant, bubble stays visible.
- The host needs `group` AND a positioning context; raise the hovered host's z-index (`hover:z-20 focus-within:z-20`) so the bubble paints above sibling rows.
- Bubble is centered (`left-1/2 -translate-x-1/2`); don't override `left-*` in `className` (Tailwind utility-precedence conflict). Use `side` instead.
- `decorative` triggers should pair with visible/labelled data; non-decorative ones need a focusable trigger carrying `aria-describedby={id}`.
