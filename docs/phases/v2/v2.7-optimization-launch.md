# Phase v2.7 — Optimization & Launch Hardening

**Branch:** sub-branches `v2.7-*` off `v2` · **Covers:** #7 (optimization, speed, security
hardening + verification)
**Depends on:** all prior phases · **Spec:** `docs/security/SECURITY.md` (verification),
`docs/workflow/DEFINITION_OF_DONE.md` (budgets)

## Goal

Final pass before launch: hit every performance budget across the whole site, harden and
**verify** the security model (the "Go/No-Go"), and run a full accessibility + cross-device
sweep. No new features — only correctness, speed, and safety.

## Segments

### v2.7.1 — Performance (`v2.7-perf`)
- **v2.7.1.1** Site-wide bundle audit; per-route code-splitting; image/font optimization;
  ensure snapshot/CDN caching is effective. *Accept:* all routes meet JS budgets; CDN hit rates verified.
- **v2.7.1.2** Lighthouse + chrome-devtools traces on every key route; fix LCP/CLS/INP to budget.
  *Accept:* Lighthouse Perf/A11y/Best-Practices ≥ 95 on Home, Players, Draft, public Waivers/Trades.
- **v2.7.1.3** DB/query audit (no N+1; indexes; keyset pagination everywhere live). *Accept:*
  slow-query check clean.
- *Files:* `next.config.mjs`, route-level dynamic imports, `db/migrations/*` (indexes).

### v2.7.2 — Security verification — Go/No-Go (`v2.7-sec`)
- **v2.7.2.1** RLS coverage test across **all** user tables (A-vs-B). *Accept:* zero cross-user
  read/write paths.
- **v2.7.2.2** Client-bundle secret scan; header scan (CSP/HSTS/etc.); TLS/HSTS preload check.
  *Accept:* no secrets shipped; headers/TLS pass.
- **v2.7.2.3** Auth/vault audit: session fixation, CSRF, OAuth state/nonce, logout invalidation,
  vault no-plaintext-to-client, rate limits. *Accept:* `ce-security-reviewer` + `security-guidance`
  pass; documented results.
- **v2.7.2.4** "Inspector isolation" proof: demonstrate that a network inspector on user A's
  session cannot reach user B's data/credentials. *Accept:* documented test confirms isolation.
- *Files:* `tests/security/**` (new), middleware/headers, docs verification log.

### v2.7.3 — Accessibility & cross-device sweep (`v2.7-a11y`)
- **v2.7.3.1** Full axe + manual SR pass (VoiceOver/NVDA) on every route; keyboard-only;
  200% zoom; each a11y mode. *Accept:* WCAG 2.2 AA across the site; no clipped numerals.
- **v2.7.3.2** Real-device matrix (small Android, iPhone, tablet, desktop) for the war-room
  layouts. *Accept:* excellent on every size; no seams, no clipping, 60fps motion.
- *Files:* test artifacts; targeted fixes only.

### v2.7.4 — Launch prep (`v2.7-launch`)
- **v2.7.4.1** Error monitoring/logging (no PII/secrets), uptime, rollback plan, env/secret
  audit. *Accept:* monitoring live; rollback documented.
- **v2.7.4.2** Final docs review + consolidate; update README/VISION to "as shipped." *Accept:*
  docs match reality.
- *Files:* monitoring config, `README.md`, `docs/**`.

## Test / QA plan

The full `DEFINITION_OF_DONE.md` checklist applied to the **entire** site, plus the security
Go/No-Go and the accessibility/device sweep above. This phase's QA *is* the launch gate.

## Risks & mitigations

- *Late-found security gap* → run the RLS + inspector-isolation tests continuously from v2.5,
  not just here; v2.7 is verification, not first discovery.
- *Perf regressions from accumulated features* → CI perf budgets from v2.1 catch most early.

## Definition of done (phase / v2 release)

8-step finish → tag `v2.7.0` → merge `v2` → **`main`** → GitHub Release for **v2.0.0 (the v2
line shipped)** → delete branches → final doc consolidation → `docs/brainstorming/v2.7-ideas.md`
+ a v2 retrospective.
