# Phase v2.0 — Foundation & De-Andrew-ification

**Branch:** `v2` (phase) · segment sub-branches `v2.0-*`
**Covers:** #8 (remove Andrew-ification) + workflow setup, design tokens, a11y/motion scaffold
**Depends on:** nothing (first phase) · **Unblocks:** everything

## Goal

Stand up the v2 working structure and the cross-cutting foundations every later phase reuses:
the design-token system, accessibility scaffolding, the motion toolkit, and a product cleansed
of personal identity. No user-facing feature work — this is the spine.

## Segments

## Test / QA plan

- Build + typecheck green; axe smoke clean on the shell.
- Theme + a11y controls manual matrix (dark/light × each a11y mode).
- Reduced-motion verified on the sample Rive + Lenis.
- Grep/CI check that fails if "Andrew" (and other personal tokens) reappear in product copy.

## Risks & mitigations

- *OKLCH browser support* → supported in all current evergreen browsers; provide an sRGB
  fallback layer for very old ones (progressive enhancement).
- *Token migration churn* → do it once here so later phases never touch color literals.
- *Rive learning curve* → start with one simple instrument; expand in v2.1.

## Definition of done (phase)

Run the 8-step finish: full QA across themes/a11y modes → `/code-review` of `main...v2` →
commit → merge `v2` → tag `v2.0.0` → delete branches → doc review → archive plan to
`docs/archive/v2/` → write `docs/brainstorming/v2.0-ideas.md`.
