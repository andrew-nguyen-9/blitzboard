# Phase v2.1 — Design System & Homepage Revamp

**Branch:** sub-branches `v2.1-*` off `v2` · **Covers:** #1 (homepage)
**Depends on:** v2.0 (tokens, motion, a11y) · **Unblocks:** consistent UI for all later tabs

## Goal

Turn the design guideline into real, reusable components, then rebuild the homepage so it
stops looking "vibe-coded": fix the hero's visible edges, fix numbers that don't fit their
cells, and add dynamic moments that *don't* slow the site (LCP ≤ 2.5s).

## Segments

### v2.1.1 — Core component kit (`v2.1-kit`)
Build the instrument primitives every tab reuses, against the guideline.
- **v2.1.1.1** `ValueDial` (Rive) — radial value gauge with reduced-motion static. *Accept:*
  renders a value, animates fill, snaps static under reduced-motion.
- **v2.1.1.2** `DistributionRidge` — ridgeline/violin for Monte Carlo boom/bust + a static SVG
  fallback. *Accept:* shows a distribution; degrades to a bar under reduced-motion.
- **v2.1.1.3** `StatCell`/`StatTable` — tabular-nums, fixed min-widths, `clamp()` sizing,
  stacks under 640px. *Accept:* **no digit clipping** at 360/768/1280/1920 or any text scale.
- **v2.1.1.4** `Ticker` (broadcast lower-third), `TierBadge`, `PredictabilityMeter`.
  *Accept:* each has a reduced-motion + a11y (label/role) story.
- *Files:* `frontend/components/{ValueDial,DistributionRidge,StatTable,Ticker,TierBadge,PredictabilityMeter}.tsx`.

### v2.1.2 — Homepage hero rebuild (`v2.1-hero`) — fixes "visible edges"
- **v2.1.2.1** Full-bleed hero: `100vw` media, mask-wipe/blur→sharp reveal, **no container
  seams**, gradient meets viewport edge. *Accept:* zero visible edges at every breakpoint; LCP
  element is static (image/text), heavy motion hydrates after paint.
- **v2.1.2.2** Kinetic split-text headline + magnetic CTAs (reduced-motion = plain). *Accept:*
  static fallback is clean; CTAs reachable by keyboard.
- **v2.1.2.3** "SCROLL" affordance + Lenis story scaffolding. *Accept:* works with reduced-motion off/on.
- *Files:* `frontend/app/page.tsx`, hero components, `mocks/v2-research/findings.md` (reference).

### v2.1.3 — Homepage data bands (`v2.1-bands`) — fixes "numbers don't fit"
- **v2.1.3.1** Scoreboard stat band using `StatCell` + `CountUp` (reduced-motion shows final
  number). *Accept:* numbers fit, align, count up smoothly, never reflow (CLS≈0).
- **v2.1.3.2** Tilt-card deck / trending ticker, GSAP per-route lazy. *Accept:* additive motion,
  not on the critical path; static fallback.
- **v2.1.3.3** Anticipatory prefetch of primary tool routes on CTA hover/focus. *Accept:*
  navigation feels instant; no over-fetch on touch.
- *Files:* `frontend/app/page.tsx`, `components/{Marquee,TiltCard,motion}.tsx`.

### v2.1.4 — Perf & polish pass (`v2.1-perf`)
- **v2.1.4.1** Per-route code-split motion libs; ensure homepage JS ≤ 180KB gzip. *Accept:*
  bundle report under budget.
- **v2.1.4.2** Lighthouse + chrome-devtools trace; fix CLS/LCP/INP to budget. *Accept:*
  Lighthouse Perf/A11y/Best-Practices ≥ 95 mobile.
- *Files:* `next.config.mjs`, dynamic imports across hero/bands.

## Test / QA plan

- Responsive matrix 360/768/1280/1920 + 200% zoom; numeric-clip check; seam check on hero.
- Reduced-motion + each a11y mode; keyboard-only homepage traversal.
- Lighthouse + axe ≥ 95; bundle-size check in CI.

## Risks & mitigations

- *Motion regressing LCP* → static LCP element + post-paint hydration is mandatory; CI perf gate.
- *Component kit scope creep* → build only what the homepage + known tabs need; expand later.

## Definition of done (phase)

8-step finish → tag `v2.1.0` → `docs/brainstorming/v2.1-ideas.md`.
