# Phase v2.6 — Gated + Public League/Waivers/Trades

**Branch:** sub-branches `v2.6-*` off `v2` · **Covers:** #5 (gate league/waivers/trades),
#6 (public generic waivers/trades)
**Depends on:** v2.5 (auth/leagues), v2.2 (value) · **Spec:** `DECISIONS_V2.md` D17

## Goal

Split the in-season tools into two planes: **authenticated** (your league's Overview, Waivers,
Trades — tailored to your imported rules) and **public** (anyone can see trending waivers and
test trades with our model). Prompt unauthenticated users to connect.

## Segments

### v2.6.1 — Gating + connect prompts (`v2.6-gate`)
- **v2.6.1.1** Route guards: `/league`, authenticated `/waivers`, `/trades` require a session +
  a connected league; otherwise show a connect/login prompt (not a dead end). *Accept:* signed-out
  user sees a clear "connect your league" CTA; signed-in-no-league sees an import CTA.
- **v2.6.1.2** Re-prompt to (re)connect ESPN/Sleeper when missing/expired (ties to v2.5 vault).
  *Accept:* expired creds surface a reconnect flow.
- *Files:* `frontend/middleware.ts`, `frontend/app/(auth)/**`, connect/prompt components.

### v2.6.2 — Authenticated league tabs (`v2.6-private`)
- **v2.6.2.1** League Overview tailored to the active league's rules (standings, rosters).
  *Accept:* reflects the user's imported league; RLS-isolated.
- **v2.6.2.2** Waivers (FAAB bid recommendations) against the active rules + trending. *Accept:*
  bids scale to remaining budget × trend for that league.
- **v2.6.2.3** Trades (Pareto-improving, need-aware) within the user's league rosters. *Accept:*
  proposals use that league's value layer; both-sides deltas + fairness shown.
- *Files:* `frontend/app/league/**`, `app/waivers/**`, `app/trades/**`, `lib/trade.ts`, `lib/queries.auth.ts`.

### v2.6.3 — Public generic tabs (`v2.6-public`) — #6
- **v2.6.3.1** Public **Waivers**: trending adds/drops + sentiment, no league needed, served
  from the public snapshot. *Accept:* anonymous user sees league-agnostic trending.
- **v2.6.3.2** Public **Trade tester**: pick any two players → our model scores the swap (value,
  fairness, boom/bust) under a chosen generic scoring profile. *Accept:* anyone can test a
  hypothetical trade without auth.
- **v2.6.3.3** Upsell path: "connect your league for tailored bids/standings." *Accept:* clear,
  non-naggy CTA from public → auth.
- *Files:* `frontend/app/(public)/waivers/**`, `(public)/trades/**`, `lib/trade.ts` (generic mode).

## Test / QA plan

- Access matrix: signed-out / signed-in-no-league / signed-in-with-league for each tab.
- RLS isolation on the private tabs; public tabs leak no user data.
- Value correctness against active rules; FAAB bid math; trade fairness.
- A11y + responsive + reduced-motion on all new surfaces.

## Risks & mitigations

- *Leaking private data into public components* → public tabs read only the anon snapshot;
  code-review + RLS test specifically for this boundary.
- *Confusing gating UX* → every gate is a helpful prompt with a clear next step, never a wall.
