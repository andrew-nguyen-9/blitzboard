# v2.4.3 — Tune & Validate (design)

> Segment **v2.4.3** of phase v2.4 (`v2.4-tune` off `v2`). Proves the v2.4.2 additive policy
> beats baselines on the 2021–2024 backtest, ablates each component, and writes
> `docs/modeling/backtest-report.md`. Builds on the v2.4.1 harness (`pipeline/backtest/`).

## Goal

1. **Baselines to beat:** raw-VORP (always pick the highest projection) and ADP-follow (pick
   the earliest ADP). These are the standard, unambiguous comparisons. *v1 autodraft is
   intentionally not resurrected* — its policy was replaced in v2.4.2, so re-adding it only to
   lose to it is dead code with no decision value.
2. **Ablations:** run the v2 policy with one component disabled at a time — K/DEF cap off,
   bench-ceiling off, bench-injury off, replacement-walk naive (`runDepletion = 1`) — to show
   each helps.
3. **Tune:** a small grid over the highest-leverage `DEFAULT_POLICY` params; keep the config
   that maximizes mean season points-for (H2H win% as tiebreak).
4. **Report:** `docs/modeling/backtest-report.md` with the policy-vs-baseline table + ablation
   deltas, seeds and 95% CIs (the harness's `aggregate` already produces CIs).

## Mechanism (small seams, no new scoring logic)

The additive policy already takes a `PolicyParams` object, so ablation = the same policy with
one field changed. Three seams:

- `frontend/lib/draftAI.ts`: `pickForTeam(ctx, params?)` forwards params to `scoreBoard`
  (currently drops them). Backward compatible (defaults to `DEFAULT_POLICY`).
- `frontend/lib/snakeDraft.ts`: `SnakeOpts.chooser?: (ctx: AIContext) => PlayerWithValue | null`
  — `runSnakeDraft` calls `opts.chooser ?? pickForTeam`. The live board (`simulate.ts`) is
  unaffected (no chooser → default policy).
- `frontend/scripts/draftSim.ts`: select the chooser from `cfg.policy` and an optional
  `cfg.params` override:
  - `"v2"` → `(ctx) => pickForTeam(ctx, cfg.params)` (params present ⇒ an ablation run)
  - `"rawvorp"` → pick max `proj(p)` in the pool
  - `"adp"` → pick min `value.adp` in the pool (earliest ADP), nulls last

Python side:
- `sim_bridge.run_draft(players, seed, num_teams, policy, params=None)` threads `params` into
  the JSON payload.
- `pipeline/backtest/tune.py` (new): drives baseline comparison, the ablation set, and the
  grid; calls the existing `run.py` scoring helpers (`season_actuals`, `_build_pool`,
  `weekly_team_points`, `aggregate`, `h2h_record`). Emits the markdown report.

## Compute & data

Network + venv fetch deps are available (verified). Missing seasons (2021/2022/2024 pbp+weekly,
2022/2024 ADP) are fetched and disk-cached by the existing `cache.py` path on first run; 2023 is
already cached. Seeds and grid size are CLI-capped to keep the sweep bounded (the v2.4.1 risks
doc: "cap seeds with variance reporting"). Default sweep: seasons 2021–2024, 6 seeds, a ≤12-point
grid — runnable in one session; widen via flags for a fuller run.

## Acceptance

- v2 policy mean season points-for and H2H win% exceed both baselines, with CIs, in the report.
- Each ablation shows a non-positive delta vs the full policy (or is flagged if not — an honest
  finding, not a hidden one).
- `npm test` (TS seams) + `pytest pipeline/tests/test_backtest_*` green; report committed.
