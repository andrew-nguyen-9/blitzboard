# Phase v2.3 — Player Data Layer & Players Tab

**Branch:** sub-branches `v2.3-*` off `v2` · **Covers:** #2 (500 cap, efficient transfer, revamp)
**Depends on:** v2.0 (kit/tokens), v2.2 (value layer) · **Unblocks:** Draft (v2.4)
**Spec:** `docs/architecture/DATA_TRANSFER.md`

## Goal

Deliver the entire player universe (no 500 cap) effortlessly: precomputed CDN snapshots,
client-side virtualization/sort/filter, lazy + prefetched detail — then rebuild the Players
tab on the v2 component kit.

## Segments

### v2.3.1 — Snapshot publisher (`v2.3-snapshot`)
- **v2.3.1.1** `pipeline/publish_snapshot.py`: emit core-list payload per `(profile×engine)`,
  brotli-compressed, content-hashed, uploaded to Storage/CDN with immutable cache headers.
  *Accept:* a snapshot for the example profile builds + uploads; idempotent re-run.
- **v2.3.1.2** `manifest.json` (short TTL) mapping `profile+engine → url+hash+count`. *Accept:*
  client can resolve current snapshot from the manifest.
- **v2.3.1.3** Choose wire format vs. 60KB budget (columnar/typed-array vs. minified-JSON).
  *Accept:* core payload ≤ 60KB for the full universe after brotli; decode verified.
- *Files:* `pipeline/publish_snapshot.py` (new), `.github/workflows/etl_daily.yml` (publish step).

### v2.3.2 — Client data layer (`v2.3-client`)
- **v2.3.2.1** Fetch manifest→snapshot, decode into an in-memory table model; cache in the
  client. *Accept:* full universe loads; most loads hit the CDN, not the DB.
- **v2.3.2.2** Virtualized rows (`@tanstack/virtual`/`virtua`). *Accept:* 2–3k rows scroll at
  60fps with a small DOM.
- **v2.3.2.3** Local sort/filter/search over the in-memory set. *Accept:* < 50ms; no round trips.
- **v2.3.2.4** Remove the v1 `.limit()`/single-`.range()` cap; keyset pagination helper for any
  remaining live queries. *Accept:* no implicit row cap anywhere.
- *Files:* `frontend/lib/queries.ts`, `frontend/lib/snapshot.ts` (new), `package.json`.

### v2.3.3 — Players tab revamp (`v2.3-ui`)
- **v2.3.3.1** Rebuild `PlayerTable` on `StatTable` + virtualization + tokens. *Accept:* no
  digit clipping; responsive stacks under 640px; sortable headers keyboard-accessible.
- **v2.3.3.2** Filters (position, team, tier, predictability, value range), instant. *Accept:*
  filter combinations update the virtual list instantly.
- **v2.3.3.3** Lazy player detail + **prefetch on hover/focus** (warm route + detail payload).
  *Accept:* opening a card feels instant; no over-fetch on touch.
- **v2.3.3.4** Player card uses `ValueDial`, `DistributionRidge`, `PredictabilityMeter`, trend
  sparkline. *Accept:* instrument readout per the guideline; a11y + reduced-motion.
- *Files:* `frontend/components/PlayerTable.tsx`, `frontend/app/players/**`.

## Test / QA plan

- Load the full universe; assert count > old 500 cap; payload ≤ 60KB; CDN cache hit verified.
- 60fps scroll (devtools trace); sort/filter/search latency; prefetch behavior on hover vs touch.
- Responsive + a11y + reduced-motion matrix; numeric-clip check.
- Snapshot publisher idempotency + manifest correctness.

## Risks & mitigations

- *Columnar decode complexity* → ship minified-JSON+brotli first if it meets 60KB; upgrade to
  columnar only if needed.
- *Stale snapshot after cron* → content-hashed URLs + short-TTL manifest make new data a new URL.
- *Memory of full dataset on low-end phones* → typed arrays keep it lean; only visible rows in DOM.
