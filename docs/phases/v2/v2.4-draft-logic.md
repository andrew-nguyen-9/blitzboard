# Phase v2.4 — Draft Logic Redo + Backtesting (2021–2025)

**Branch:** sub-branches `v2.4-*` off `v2` · **Covers:** #4 (draft logic, bench, K/DEF hoarding)
**Depends on:** v2.2 (value), v2.3 (data) · **Spec:** `docs/modeling/DRAFT_LOGIC.md`

## Goal

Rebuild the draft policy around starting-lineup marginal value + an explicit bench-upside
model + a K/DEF cap, and **prove it on 2021–2025** with a reproducible backtest harness. Manual
and synced boards consume the same policy.

## Segments

### v2.4.1 — Backtest harness (`v2.4-harness`)
- **v2.4.1.1** Assemble 2021–2025 nflverse actuals scored under league rules (superflex,
  half-PPR, distance-K, yardage-D/ST). *Accept:* per-season per-player actual fantasy points reproducible.
- **v2.4.1.2** Draft simulator: 12-team superflex snake, realistic ADP pool, pluggable policy.
  *Accept:* runs a full mock draft deterministically per seed.
- **v2.4.1.3** Evaluation: weekly greedy optimal lineup from actuals → season points-for + H2H
  vs. field, averaged over seeds with CIs. *Accept:* produces a comparable score per policy.
- *Files:* `pipeline/backtest/**` (new), `pipeline/history_ingest.py` (reuse).

### v2.4.2 — Draft policy (`v2.4-policy`)
- **v2.4.2.1** `marginal_starter_value` against the optimal starting lineup (incl. FLEX/OP).
  *Accept:* a redundant 3rd RB scores low; a needed superflex QB scores high.
- **v2.4.2.2** **Bench model**: upside (ceiling), handcuff/cover correlation, bye/injury
  redundancy → `expected_starts × value_when_started × availability`. *Accept:* a high-ceiling
  WR4 beats a 2nd K/DEF for a bench slot in tests.
- **v2.4.2.3** K/DEF cap (≤1 each until final rounds) + positional caps + superflex QB-run
  model. *Accept:* bots stop early K/DEF hoarding; QB runs modeled.
- *Files:* `frontend/lib/draftAI.ts`, `frontend/lib/draft.ts`, shared policy module + tests.

### v2.4.3 — Tune & validate (`v2.4-tune`)
- **v2.4.3.1** Grid/Bayesian tune `f(ρ)` exponent, bench weights, scarcity curve on backtest.
  *Accept:* tuned params beat v1 + raw-VORP + ADP-follow on mean points-for + H2H (report).
- **v2.4.3.2** Ablations (drop discount / bench-upside / cap individually). *Accept:* each
  component shown to help; written to `docs/modeling/backtest-report.md`.
- *Files:* `pipeline/backtest/**`, `docs/modeling/backtest-report.md` (generated).

### v2.4.4 — Wire into the boards (`v2.4-boards`)
- **v2.4.4.1** Live (Sleeper/ESPN sync) + offline manual boards consume the shared policy.
  *Accept:* identical recommendations across input sources (D7).
- **v2.4.4.2** Draft UI surfaces *why* a pick is suggested (need/upside/scarcity), with the
  v2.1 kit. *Accept:* recommendation rationale legible; reduced-motion + a11y.
- *Files:* `frontend/components/DraftRoom.tsx`, `DraftAnalysis.tsx`, `useSleeperSync.ts`, `useEspnSync.ts`.

## Test / QA plan

- Harness reproducibility (fixed seeds); policy unit tests on synthetic boards.
- Backtest report: v2 > baselines with CIs; K/DEF not drafted early; benches upside-heavy.
- Board parity test (manual vs synced give same picks); UI a11y/reduced-motion.

## Risks & mitigations

- *Overfitting to 2021–2025* → hold out a season for validation; prefer simple, explainable
  terms; report CIs not point wins.
- *ADP reconstruction fidelity* → use multiple ADP sources / sensitivity-test the pool.
- *Compute time* → vectorize the sim; cap seeds with variance reporting.

## Definition of done (phase)

8-step finish → tag `v2.4.0` → `docs/brainstorming/v2.4-ideas.md`. Backtest report committed.
