# v2.3 — ideas & follow-ups for the backlog

Captured during the Player Data Layer & Players-tab phase. Not committed work — candidates
for later phases. (Carries forward the still-open items from `v2.2-ideas.md` and earlier.)

## Delivery / pipeline

- **Publish the `monte_carlo` snapshot too.** The cron only runs `value_engine_run.py
  --engine vorp`, so only the vorp snapshot is published; the Players engine toggle to
  Monte Carlo falls back to the live query / empty state. Add an MC engine run + publish
  step so the toggle is fully snapshot-served. (Owner: whichever phase runs MC in cron.)
- **Snapshot freshness monitor.** Content-hashed URLs + short-TTL manifest mean a stale
  snapshot is invisible. Add a synthetic check (manifest age, count sanity) — natural fit
  for the v2.7 launch monitoring (Sentry/uptime).
- **Edge cache-hit verification.** Persisted `cache-control: max-age` and CDN cache-hit are
  deploy-verified only; fold a one-time check into the v2.7 perf pass.
- **Columnar / typed-array upgrade — not yet needed.** 2,800 players = 45.4KB gzip, well
  under 60KB. Revisit only if the universe or column set grows enough to miss the budget
  (the wire format is versioned via `payload.v`, so a bump is clean).
- **Wire-version negotiation.** `decodeSnapshot` maps by column name (tolerant of added
  columns) but doesn't branch on `payload.v`. Add explicit handling before the first
  breaking format change so old clients degrade gracefully.

## Frontend / UX

- **Persist filters in the URL.** Search/position/team/tier/ρ are local state; encoding
  them in query params makes views shareable and survives reloads (pairs with the existing
  `?engine=` param).
- **"Jump to rank / player" affordance.** With 2,800 virtualized rows, a quick jump or
  keyboard "g + rank" beats scrolling; also helps SR users who only perceive the rendered
  window.
- **Multi-select position + value-range slider.** Shipped a single-position toggle + a ρ
  slider; a value/VOR range and multi-position OR-filter were in the plan's spirit.
- **Prewarm the snapshot.** Kick `loadSnapshot()` on app shell mount (or via a tiny
  `<link rel="preload">` to the manifest) so the Players tab is instant on navigation.
- **Restore the server-rendered count.** The page lost its SSR "{N} players" line (data is
  now client-loaded). A tiny manifest read at the server could render the count for faster
  perceived load.

## Accessibility (→ v2.7-a11y)

- **Custom-cursor label contrast.** `.cursor-label` uses `color: var(--bg-0)` (light in the
  light theme) on the accent-pink hover ring → axe color-contrast 2.06:1. The
  `mix-blend-mode: difference` on `.cursor-ring` provides the *visual* contrast (axe can't
  model it), so this is largely a false positive — but the same `text-bg`-on-accent pattern
  recurs in EngineToggle / position pills and deserves a real audit in the light theme.
  Pre-existing; surfaced by the Players-tab QA. Fix globally in v2.7-a11y.
- **Virtualized-table SR pass.** `role="table/rowgroup/row/columnheader/cell"` +
  `aria-rowcount`/`aria-rowindex` are in place, but a real screen-reader walk (VoiceOver/
  NVDA) of a windowed table is a v2.7-a11y deliverable.

## Engineering hygiene (carry-forward, still open)

- **Pipeline tests aren't in CI.** `pipeline/tests/*` + `selftest.py` (now including
  `test_snapshot.py`) run only locally; CI covers only the frontend. The snapshot
  publisher has zero CI coverage. Add a Python test job to `ci.yml`. (Open since v2.0.)
- **Flip the axe smoke to a blocking gate** + add `/players` (and other key routes) to its
  route list; add required status checks to branch protection. (Open since v2.0.)
- **Unify player routing on `sleeper_id`.** Detail now resolves UUID *or* sleeper_id; a
  later cleanup could migrate the draft/trade/roster boards' UUID links to sleeper_id so
  there's one canonical public id (and the dual-resolution branch can retire).
