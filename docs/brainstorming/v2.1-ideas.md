# v2.1 — ideas & follow-ups for the backlog

Captured during the Design System & Homepage phase. Not committed work — candidates
for later phases. (Carries forward the still-open items from `v2.0-ideas.md`.)

## Nav — surfaced by homepage QA (own segment, likely a small "chrome" pass)

- **XL text-scale overflow at 768px.** With the A11ySettings text size at **XL** on a
  ~768px viewport, the Nav's right-hand controls ("Access" / "Auto" pills) overflow the
  viewport by ~21px (horizontal scroll). The hero itself passes XL at every breakpoint;
  this is purely `components/Nav.tsx` fixed-pill layout. Make the controls wrap / shrink
  or move into the A11y menu under a width threshold.
- **Eager prefetch over-fetches on touch.** The Nav links use Next's default `<Link>`
  prefetch, which warms all five tool routes on load **including on touch devices**
  (they're in the viewport). The homepage CTAs/tiles now use intent-based prefetch
  (`usePrefetchOnIntent`, `prefetch={false}`); apply the same to the Nav so touch users
  don't fetch the whole app on every page load. (Bonus: it makes the homepage's intent
  prefetch non-redundant — today the Nav already warms those routes.)

## Homepage motion — the deferred "wow"

- **GSAP scroll-story.** v2.1.2.3 shipped the Lenis scaffolding (`lib/lenis.ts` shared
  instance, `SmoothScroll` registers it) and `lib/gsap.ts` lazy loader, but no actual
  ScrollTrigger choreography yet. Build the pinned/parallax homepage story on top of
  this (per-route lazy, reduced-motion gated at the call site).
- **Bespoke homepage Rive instrument.** Still open from `v2.0-ideas.md`: place a real
  `<RiveInstrument>` (e.g. a value dial or sentiment pulse) in the hero/deck, replacing
  the public `sample.riv`, and verify the reduced-motion rest frame in situ.
- **Route-scope Lenis.** Still open from v2.0: Lenis inits on every route after paint.
  Now that there's a scroll-story scaffold, consider initializing it only on routes that
  have one.

## Perf — the LCP tail

- **`display: optional` for the body font.** Mobile Lighthouse Perf is 95–96; the only
  soft metric is simulated LCP (2.7–2.9s), gated by the Hanken body-font swap on the hero
  subhead (the LCP element). `display: "optional"` would pin LCP to first paint (metric-
  matched fallback, no late swap) and likely push Perf to ~99 — but it's a site-wide body
  font, so some slow first-loads would never see Hanken. Worth A/B-ing against the design
  bar before taking it.
- **Flip the axe smoke to a blocking gate.** Still open from v2.0: the CI `axe smoke` job
  is green on every v2.1 PR but still `continue-on-error`. Remove that line.

## Component kit polish

- **`StatCell count` + `sign`.** The count-up path renders via `CountUp`, which omits the
  `+`/`−` sign that `formatStat` adds, so a counting cell with `sign` reserves width for
  the sign but never shows it (harmless over-reservation, unused on the homepage). If a
  signed counting stat ever ships, thread `sign` through `CountUp`.
- **Count-up and screen readers.** Below-fold `CountUp` cells render `0` until scrolled
  into view (then tally up), so a screen-reader virtual-cursor pass before scrolling reads
  "0 Players ranked". Reduced-motion users already get the final number immediately;
  consider an `aria-hidden` animated value + a visually-hidden final value, or only arming
  the count when `inView` is imminent, when we next touch the kit.
