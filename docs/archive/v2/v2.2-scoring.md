# Phase v2.2 — Scoring & Value-Engine Upgrade

**Branch:** sub-branches `v2.2-*` off `v2` · **Covers:** #3 (K/DEF overvalued)
**Depends on:** v2.0 · **Unblocks:** Players (v2.3), Draft (v2.4), Trades/Waivers (v2.6)
**Spec:** `docs/modeling/SCORING.md`, `docs/modeling/VALUE_ENGINE.md`

## Goal

Make the value layer honest: discount value by predictability and set K/DEF replacement at
streamer level so kickers and defenses stop being overvalued — without regressing
QB/RB/WR/TE. Surface the *why* (predictability) in the UI.

## Segments

### v2.2.1 — Predictability scoring (`v2.2-predictability`)
- **v2.2.1.1** Add `predictability` to the `Projector` output: year-over-year positional rank
  stability + own variance + TD/turnover share. *Accept:* every player row has `ρ∈[0,1]`;
  K/DEF skew low, stable RB/WR high (sanity check vs. history).
- **v2.2.1.2** Persist `predictability` to `projections`/`player_value`; expose in queries.
  *Accept:* value rows carry it; snapshot schema updated.
- *Files:* `pipeline/models/*projector*.py`, `pipeline/value_engine_run.py`, `db/migrations/*`.

### v2.2.2 — Discounted VORP + streamer replacement (`v2.2-vorp`)
- **v2.2.2.1** Implement `adj_value = replacement + raw_VORP * f(ρ)` in `VorpEngine`. *Accept:*
  parameter `k` configurable; unit tests on synthetic players.
- **v2.2.2.2** K/DEF replacement = weekly-streamer percentile (not 12th). *Accept:* K/DEF VORP
  spread collapses; best K/DEF ranks in last-rounds range.
- **v2.2.2.3** Keep bounded league signal (distance-K, yardage-D/ST via existing K/DEF
  projectors). *Accept:* big-leg kickers / stout defenses edge ahead *within* the compressed band.
- *Files:* `pipeline/value_engine_run.py`, `frontend/lib/score.ts` (if any client mirror), tests.

### v2.2.3 — Monte Carlo alignment (`v2.2-mc`)
- **v2.2.3.1** Feed predictability-aware distributions into `MonteCarloEngine` (vectorized,
  from v1 P7). *Accept:* volatile players get correctly wide boom/bust; calibration improves.
- **v2.2.3.2** Reliability check: realized boom/bust vs. emitted distribution. *Accept:*
  reliability diagram within tolerance; documented.
- *Files:* `pipeline/models/montecarlo*.py`, a calibration script.

### v2.2.4 — Surface the why (`v2.2-ui`)
- **v2.2.4.1** Wire `PredictabilityMeter` (from v2.1 kit) into the player card. *Accept:* a
  low-value K/DEF visibly explains itself; a11y label present.
- **v2.2.4.2** Engine-toggle viz morph (dial⇄ridgeline) reflects the new distributions.
  *Accept:* toggling re-ranks the board and morphs the viz.
- *Files:* `frontend/components/{PredictabilityMeter,ValueDial,DistributionRidge,EngineToggle}.tsx`, player route.

## Test / QA plan

- Unit tests: `f(ρ)` monotonicity, replacement-from-demand, streamer percentile.
- Backtest sanity (full harness in v2.4): K/DEF land in last-rounds range across 2021–2025;
  no QB/RB/WR/TE regression vs. v1.
- Calibration/reliability check documented.
- UI: predictability legible, reduced-motion + a11y on new viz.

## Risks & mitigations

- *Over-compressing a genuinely elite K/DEF* → bounded league signal + tuned `k`; backtest gate.
- *Predictability signal noisy with sparse history* → shrink toward positional prior for
  low-sample players.

## Definition of done (phase)

8-step finish → tag `v2.2.0` → `docs/brainstorming/v2.2-ideas.md`. Hand the tuned value layer
to v2.3/v2.4.

## As shipped (v2.2.0)

Archived at phase close. All four segments merged to `main` (PRs #13–#16). Every
pipeline suite green (`test_predictability` 6, `test_vorp_discount` 5, `test_calibration`
4, `test_mc` 3, `selftest`, `calibration_check`); frontend `tsc` + `vitest` 34 + `build`
clean; the player card is axe-clean (0 serious/critical) in both engine modes across
360/1280 and reduced-motion. Deltas from the plan above:

- **v2.2.1 predictability** — shipped `models/predictability.py`: ρ blends a positional
  YoY rank-stability prior (measured Pearson-on-ranks, blended with a research-grounded
  structural prior) + the player's own season-to-season CV + TD/turnover share, then
  **shrinks toward the prior by sample size** (`w=n/(n+τ)`). Computed once per player in
  the orchestrator and persisted to **both** `projections` and `player_value` (migration
  `20260625_v2.2.1_add_predictability.sql` + `schema.sql`); exposed through `queries.ts`/
  `types.ts`. **Plan note:** ρ is attached at the orchestrator (which owns the shared
  store) rather than inside each projector, so all four projector families carry it with
  no constructor plumbing.
- **v2.2.2 vorp** — `f(ρ)=ρ^k` (configurable `discount_k`) scales the *positive value
  premium*; raw `vor`/`boom`/`bust` stay undiscounted for interpretable lineup math, so
  only the board-ranking `value` carries the discount. K/DEF replacement = the upper-
  middle weekly-streamer percentile (`STREAMER_PCT`). **No-regression guard proved as a
  unit test:** under uniform ρ the discount is a constant factor → offensive order is
  identical to undiscounted. Best K/DST lands ~rank 80/96 in the fixture; a big-leg K
  still edges a weak one within the compressed band.
- **v2.2.3 mc** — MC σ widened by `1+MC_VOL_GAIN·(1−ρ)` with clip bounds re-derived from
  the *effective* σ (else the widening is clipped away). Replacement (incl. the K/DEF
  streamer level) factored into a **shared `_replacement_index`** so the VORP↔MC toggle
  prices the board coherently; added a `seed` for reproducible runs. The reliability
  check (`models/calibration.py`, PIT→KS) + `calibration_check.py` demonstrate the
  widening flattens the diagram (error 0.084→0.026; volatile cohort 0.120→0.036),
  documented in `docs/modeling/calibration.md`.
- **v2.2.4 ui** — `PredictabilityMeter` on the player card (hidden when ρ is absent);
  the engine toggle morphs the instrument VORP-dial ⇄ MC-ridgeline. The ridge is rebuilt
  from the stored P90/P10 spread via `gaussianSamples` (Acklam probit at evenly-spaced
  quantiles → deterministic, **SSR-safe, no RNG/hydration drift**). The Players board
  already re-ranked by engine, so v2.2.4.2's "re-rank" needed no board change.

**Tunables left configurable for the v2.4 backtest (per plan):** `DISCOUNT_K`,
`STREAMER_PCT`, `MC_VOL_GAIN`, and the predictability weights/prior — none hand-tuned to a
target ranking.

**Deploy dependency (migrate-first):** the client + pipeline both reference
`predictability`; `db/migrations/20260625_v2.2.1_add_predictability.sql` (additive,
idempotent) must be applied to the live Supabase **before** the production deploy, else
the players board degrades to empty until the column exists.

**Deferred (logged in `docs/brainstorming/v2.2-ideas.md`):** pipeline tests aren't in CI;
the populated-card-with-live-data view awaits the migration + a `monte_carlo` pipeline run;
final numeric tuning of all knobs is the v2.4.3 backtest gate.
