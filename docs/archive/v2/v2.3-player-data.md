# Phase v2.3 — Player Data Layer & Players Tab

**Branch:** sub-branches `v2.3-*` off `v2` · **Covers:** #2 (500 cap, efficient transfer, revamp)
**Depends on:** v2.0 (kit/tokens), v2.2 (value layer) · **Unblocks:** Draft (v2.4)
**Spec:** `docs/architecture/DATA_TRANSFER.md`

## Goal

Deliver the entire player universe (no 500 cap) effortlessly: precomputed CDN snapshots,
client-side virtualization/sort/filter, lazy + prefetched detail — then rebuild the Players
tab on the v2 component kit.

## Segments

### v2.3.1 — Snapshot publisher (`v2.3-snapshot`)
- **v2.3.1.1** `pipeline/publish_snapshot.py`: emit core-list payload per `(profile×engine)`,
  brotli-compressed, content-hashed, uploaded to Storage/CDN with immutable cache headers.
  *Accept:* a snapshot for the example profile builds + uploads; idempotent re-run.
- **v2.3.1.2** `manifest.json` (short TTL) mapping `profile+engine → url+hash+count`. *Accept:*
  client can resolve current snapshot from the manifest.
- **v2.3.1.3** Choose wire format vs. 60KB budget (columnar/typed-array vs. minified-JSON).
  *Accept:* core payload ≤ 60KB for the full universe after brotli; decode verified.
- *Files:* `pipeline/publish_snapshot.py` (new), `.github/workflows/etl_daily.yml` (publish step).

### v2.3.2 — Client data layer (`v2.3-client`)
- **v2.3.2.1** Fetch manifest→snapshot, decode into an in-memory table model; cache in the
  client. *Accept:* full universe loads; most loads hit the CDN, not the DB.
- **v2.3.2.2** Virtualized rows (`@tanstack/virtual`/`virtua`). *Accept:* 2–3k rows scroll at
  60fps with a small DOM.
- **v2.3.2.3** Local sort/filter/search over the in-memory set. *Accept:* < 50ms; no round trips.
- **v2.3.2.4** Remove the v1 `.limit()`/single-`.range()` cap; keyset pagination helper for any
  remaining live queries. *Accept:* no implicit row cap anywhere.
- *Files:* `frontend/lib/queries.ts`, `frontend/lib/snapshot.ts` (new), `package.json`.

### v2.3.3 — Players tab revamp (`v2.3-ui`)
- **v2.3.3.1** Rebuild `PlayerTable` on `StatTable` + virtualization + tokens. *Accept:* no
  digit clipping; responsive stacks under 640px; sortable headers keyboard-accessible.
- **v2.3.3.2** Filters (position, team, tier, predictability, value range), instant. *Accept:*
  filter combinations update the virtual list instantly.
- **v2.3.3.3** Lazy player detail + **prefetch on hover/focus** (warm route + detail payload).
  *Accept:* opening a card feels instant; no over-fetch on touch.
- **v2.3.3.4** Player card uses `ValueDial`, `DistributionRidge`, `PredictabilityMeter`, trend
  sparkline. *Accept:* instrument readout per the guideline; a11y + reduced-motion.
- *Files:* `frontend/components/PlayerTable.tsx`, `frontend/app/players/**`.

## Test / QA plan

- Load the full universe; assert count > old 500 cap; payload ≤ 60KB; CDN cache hit verified.
- 60fps scroll (devtools trace); sort/filter/search latency; prefetch behavior on hover vs touch.
- Responsive + a11y + reduced-motion matrix; numeric-clip check.
- Snapshot publisher idempotency + manifest correctness.

## Risks & mitigations

- *Columnar decode complexity* → ship minified-JSON+brotli first if it meets 60KB; upgrade to
  columnar only if needed.
- *Stale snapshot after cron* → content-hashed URLs + short-TTL manifest make new data a new URL.
- *Memory of full dataset on low-end phones* → typed arrays keep it lean; only visible rows in DOM.

## Definition of done (phase)

8-step finish → tag `v2.3.0` → `docs/brainstorming/v2.3-ideas.md`.

## As shipped (v2.3.0)

Delivered across PRs #18–#20 (`v2.3-snapshot`, `v2.3-client`, `v2.3-ui`). The plan held;
the deltas below are the engineering decisions made against reality.

- **v2.3.1 snapshot** (`pipeline/publish_snapshot.py`) — per `(profile × engine)` emits a
  **compact array-of-arrays** payload keyed by a short column header
  (`sid,n,pos,tm,val,vor,rnk,boom,bust,rho,trend`), where **`sid` is the Sleeper id, not
  the UUID** — 2,800 UUIDs carry ~45KB of irreducible high-entropy bytes that blow the
  budget. **Wire format = gzip, not brotli** (the planned default): `/code-review` found
  that Supabase Storage (via storage3) does **not** persist a `Content-Encoding` header and
  mangles a full `cache-control` directive into `max-age=<value>`. So the blob is stored
  octet-stream with `cache-control = <seconds>`, and the **client decodes gzip natively
  with `DecompressionStream('gzip')`** (brotli isn't supported there; gzip is) — no JS
  decompression dep, no edge-compression dependency. Idempotent via deterministic gzip
  (`mtime=0`) → content hash = object name. **Budget proven empirically: 2,800-player
  synthetic universe = 45.4KB gzip ≤ 60KB** (v2.3.1.3 satisfied without going columnar).
  `manifest.json` (short TTL) maps `profile:engine → url+hash+count`.
- **v2.3.2 client** (`frontend/lib/snapshot.ts`) — resolve manifest → fetch immutable blob
  → `gunzipJson` → `decodeSnapshot` **by column name** (resilient to column reorder),
  cached per immutable URL; null-safe fallback to live query / empty state. `queries.ts`
  `getAllPlayersByValue` moved from offset `.range()` to **keyset pagination on `rank`**
  (no implicit row cap; v2.3.2.4). `@tanstack/react-virtual` added (decision #5).
- **v2.3.3 ui** (`components/PlayerTable.tsx`) — client island over the snapshot;
  virtualized rows (~26 in DOM for 900); instant filters (search, position, team, tier,
  ρ≥); sortable `<button>` headers with `aria-sort` on `role="columnheader"`, plus
  `role="rowgroup"` + `aria-rowindex`; mono/`tabular-nums` + `ch`-width (no clipping);
  columns collapse <640px. Rows link by `sleeper_id`, prefetch on intent. The page is now
  a thin server shell. **`getPlayerDetail` resolves by UUID *or* sleeper_id**, so existing
  UUID links (draft/trade/roster boards) and snapshot links both work. `tierMap`
  generalized to `{id,position,value}`. The card instruments (v2.2.4) are unchanged and
  now receive the snapshot links — v2.3.3.4 needed no card rebuild.

**Scope move:** the league-aware **free-agent filter** was removed from the (now public)
Players tab — it depends on league rosters and belongs to the gated league view (**v2.6**).

**Deploy dependency (migrate-first):** `db/migrations/20260625_v2.3.1_snapshots_bucket.sql`
(public `snapshots` bucket + anon-read policy) must be applied to live Supabase **before**
the production deploy, **and** the pipeline must run `publish_snapshot.py` once, or the
Players tab degrades to its empty state until a snapshot exists. The rest of the app
(draft/trades) is unaffected (still live-DB via the keyset query).

**Verified locally:** all pipeline tests + selftest; frontend build/typecheck/lint/37 unit
tests; Playwright route-mock QA of the populated table (virtualization, filters, sort,
sleeper_id links, axe-clean table markup). **Deploy-verified (needs the bucket + a pipeline
run):** live upload, persisted `cache-control`, edge cache-hit, 60fps trace on real data.

**Deferred (logged in `docs/brainstorming/v2.3-ideas.md`):** pipeline tests still aren't in
CI; the decorative custom-cursor label trips axe color-contrast in the light theme
(`mix-blend-mode` false-positive, pre-existing → v2.7-a11y); columnar/typed-array upgrade
unneeded at current budget.
