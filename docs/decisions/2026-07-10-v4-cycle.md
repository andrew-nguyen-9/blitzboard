### Factor framework (F3, E1, E3)
One file per factor under `pipeline/models/factors/`, auto-discovered — a new factor is a new
file subclassing `Factor` (`compute(ctx)→float`, identity = 1.0 mult / 0.0 delta), **zero**
`projector.py` edit. Composition: `mean' = mean*PROD(mult) + SUM(delta)`, ±1.28σ preserved.
Every factor reads only `ctx.metadata`; absent data → identity (clamped bands), so no historical
backtest regresses. Ship a WIP factor `enabled=False`.
**Why:** additive, degrade-safe, backtest-provable per factor (byte-identical vs `factors=[]`).
Factors are orthogonal to predictability `f(ρ)` (applied later on VALUE) — do not double-count.

### Injury applied proj-proportional, not a multiply (E1)
`score -= (1-avail)*max(0,proj)`, not a flat value multiply.
**Why:** VONA zeroes the healthy peer's marginal, so a multiply let a high-proj IR stud win.

### K/DEF value tune (E1)
`DISCOUNT_K` 1.0→1.2, `STREAMER_PCT` 0.60→0.62 → every real starter out-ranks the best K/DST,
offense order preserved under uniform ρ. `PlayerValue.vor` stays the raw linear lineup-sum unit;
`value` is the shaped/tuned draft value. Trade math (E7) reads `value` → inherits the tune, no
E7 code change.

### Autodraft invariant (E1)
No team ends with an empty startable offensive slot; ≤1 K & ≤1 DST before the final 2 rounds.
Fixture-asserted across 4 seeds.

### Search: client-side Bloom pre-check (E9a)
FNV-1a double-hash trigram membership; no false negatives (internal 3-grams ⊆ pg_trgm word
trigrams), <3-char queries pass through to the prefix path.
**Why:** a miss costs 0 network (instant); a hit costs one RPC.

### Auth (E4)
OAuth "logged out" root cause = callback redirecting to the proxy-internal origin (cookies scoped
to the public host). Code fix only — Google id/secret + hCaptcha stay server-side (Supabase Auth
dashboard). Captcha verification is server-side only (`botDefense.ts`, `runtime=nodejs`) — never
move token verification client-side.

### External data + secrets (F2, E5, E6)
`pipeline/adapters/`: `Adapter` base, auto-discovered (new source = new file). `requires_key`
env absent → `run()` returns `[]` (no fetch/write/raise); idempotent. New sources are
keyless-first, degrade-graceful (no paid tiers); ETL steps are additive + `continue-on-error` so
a flaky feed never reds the run. Betting odds = the-odds-api free tier; "most-backed" proxied by
implied win probability (no popularity field).

### Design (F1, E10, E11, E12)
Neon is rationed: hover/focus-only ignite on cards, no resting neon-outline; tooltip/selection/
underline are the only always-on neon (NORTH_STAR §Do/Don't). Neon tokens are additive, derived
from `--accent-h` (re-tint per league), swapped at `[data-theme]`. Mobile 44px targets via `md:`
revert, not `pointer:coarse`. Two sub-44 targets intentionally left (footer credit + spine info-`i`)
— inline text-links, WCAG 2.5.5 exception.

## Gotchas worth remembering

- **Sticky-header-in-overflow bug (E8):** a `sticky top-14` header inside `overflow-x-auto`
  binds to that container, not the window; with a window virtualizer the container never scrolls
  vertically → header drifts off. Fix: header pinned to the window, `translateX` synced to the
  body's `scrollLeft`.
- **College→NFL join (E2):** keyed on normalized full name (CFBD carries no NFL id) — documented
  limitation; factor degrades to identity without `college_production`.
- **Serena worktree noise:** `.serena/project.yml` is modified by project activation in
  worktrees; not staged, ignore it.
